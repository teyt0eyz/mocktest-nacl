# NACL Mock Exam — คู่มือส่งต่อสำหรับ AI ที่จะสร้างเว็บไซต์ทำข้อสอบ

เอกสารนี้อธิบายไฟล์ `nacl_mocktest_data.json` และสิ่งที่เว็บไซต์ mock-test ควรทำ
ข้อมูลอ้างอิงจาก blueprint จริงของสนามสอบ NACL (https://nextpath.ce-nacl.com/blueprint)

## 1. ภาพรวมข้อมูล

- ไฟล์ `nacl_mocktest_data.json` มีข้อสอบ **5 ชุด** (`sets[0]` ถึง `sets[4]`)
- แต่ละชุดมี **130 ข้อ** แบ่งเป็น 8 หมวดตาม blueprint จริง รวม 305 คะแนน
- ประเภทคำถาม 3 แบบ: `"mc"` (ปรนัย 5 ตัวเลือก, 115 ข้อ/ชุด), `"ms"` (เลือกได้หลายคำตอบ, 10 ข้อ/ชุด), `"written"` (เติมคำตอบ/เขียนตอบ, 5 ข้อ/ชุด)
- คำถามส่วนใหญ่ในแต่ละชุดแตกต่างกัน (สุ่มจากคลังคำถามที่มีมากกว่า 130 ข้อต่อหมวดในกรณีหมวดคณิตศาสตร์/เครือข่าย ที่สร้างด้วยสูตรคำนวณอัตโนมัติ) — **ข้อยกเว้น**: หมวด "Cloud Computing" และ "ความสนใจทั่วไปด้านเทคโนโลยี" เป็นคำถามเชิงแนวคิดที่เขียนด้วยมือ คลังคำถามมีจำกัด (~45-60 ข้อ) จึงอาจมีคำถามซ้ำข้ามชุดบ้าง (ตัวเลือกคำตอบจะถูกสลับลำดับใหม่ทุกครั้งเพื่อลดการจำตำแหน่ง) ควรแจ้งผู้ใช้ตามนี้ หรือถ้าต้องการ ให้ทำ deduplication เพิ่มเติมตอนแสดงผล (เช่น ไม่แสดงคำถามซ้ำกับชุดที่ทำไปแล้วในช่วง session เดียวกัน)

## 2. โครงสร้าง JSON

```
{
  "exam_info": { name, source_reference, exam_date, result_date, total_questions, total_score, question_type_counts },
  "scoring_rules": { multiple_choice: {correct:+2, wrong:-1, blank:0}, note: "..." },
  "sets": [
    {
      "set_id": 1,
      "set_name": "ชุดที่ 1",
      "categories": [
        {
          "name": "ชื่อหมวด",
          "points_total": 20,
          "question_counts": {"mc":10, "ms":0, "written":0},
          "questions": [
            {
              "id": "Q0001",
              "type": "mc" | "ms" | "written",
              "subtopic": "หัวข้อย่อย",
              "question": "ข้อความคำถาม",
              "options": ["...", "...", "...", "...", "..."],   // เฉพาะ mc/ms
              "answer": 2,                                       // mc: index เดียว | ms: array ของ index | written: ไม่มี field นี้
              "explanation": "คำอธิบายเฉลย (บางข้ออาจว่างเปล่า)"
            }
          ]
        }
      ]
    }
  ]
}
```

หมายเหตุชนิดข้อมูลของ `answer`:
- `type: "mc"` → `answer` เป็น **integer เดียว** (index ของตัวเลือกที่ถูกต้องใน `options`)
- `type: "ms"` → `answer` เป็น **array ของ integer** (index ของตัวเลือกที่ถูกทั้งหมด) — ผู้ใช้ต้องเลือกให้ตรงครบทุกข้อจึงจะถือว่าถูก (ไม่มีระบบให้คะแนนบางส่วนที่ระบุใน blueprint ต้นทาง แนะนำให้เว็บใช้เกณฑ์ all-or-nothing เป็นค่าเริ่มต้น แต่ทำ toggle ให้ปรับได้)
- `type: "written"` → `answer` เป็น **string คำตอบที่ถูกต้อง** ควรใช้การเทียบแบบ case-insensitive และ trim ช่องว่าง เนื่องจากผู้ใช้อาจพิมพ์ตัวพิมพ์เล็ก/ใหญ่ต่างกัน (เช่น "tcp" vs "TCP")

## 3. กติกาการให้คะแนน (สำคัญ)

จากหน้า blueprint ต้นทางระบุไว้ชัดเจนเฉพาะข้อ "ปรนัย 5 ตัวเลือก" เท่านั้น:
- ตอบถูก: **+2 คะแนน**
- ตอบผิด: **-1 คะแนน** (มีการหักคะแนน!)
- ไม่ตอบ: **0 คะแนน**

**สำหรับข้อ "เลือกหลายคำตอบ" และ "เขียนตอบ" หน้า blueprint ต้นทางไม่ได้ระบุเกณฑ์คะแนนไว้ชัดเจน** ผู้ใช้ (Toey) ควรตรวจสอบกับผู้จัดสอบเพิ่มเติมถ้าเป็นไปได้ ระหว่างนี้แนะนำให้เว็บไซต์ใช้ค่า default ที่ปรับได้ เช่น ให้คะแนนแบบ all-or-nothing สำหรับข้อเลือกหลายคำตอบ และให้คะแนนเต็มเมื่อคำตอบเขียนตอบตรงกับเฉลย

## 4. ฟีเจอร์ที่แนะนำให้เว็บไซต์มี

1. **หน้าเลือกชุดข้อสอบ** — ให้ผู้ใช้เลือกทำชุดที่ 1–5 (หรือสุ่มคละจากทุกชุด)
2. **แสดงคำถามตามหมวดและ progress bar** ตามจำนวนข้อในแต่ละหมวด (130 ข้อ/ชุด)
3. **ตัวจับเวลา (Timer)** — ข้อสอบจริงน่าจะมีเวลาจำกัด ควรมีตัวเลือกให้ตั้งเวลา (ไม่ได้ระบุเวลาสอบจริงในหน้า blueprint จึงให้ผู้ใช้ตั้งเองได้)
4. **สรุปผลหลังทำเสร็จ**:
   - คะแนนรวมเทียบกับ 305 คะแนนเต็ม
   - คะแนนแยกตามหมวด (เพื่อให้เห็นจุดอ่อน/จุดแข็ง เช่น อ่อน Networks แต่แข็ง Cloud)
   - แสดงเฉลยพร้อม `explanation` ของแต่ละข้อที่ตอบผิด
5. **โหมดฝึกฝนรายหมวด** — ให้เลือกทำเฉพาะบางหมวด (เช่น ฝึกเฉพาะ Subnetting) แยกจากการทำเต็มชุด เพราะ Networks (85 คะแนน) และ Cloud (70 คะแนน) เป็น 2 หมวดที่มีน้ำหนักคะแนนสูงสุด
6. **บันทึกประวัติการทำข้อสอบ** (local storage หรือฐานข้อมูล) เพื่อดูพัฒนาการ

## 5. ข้อจำกัดที่ควรรู้ก่อนใช้งานจริง

- คำถามทั้งหมดสร้างขึ้นเพื่อ**จำลองรูปแบบและสัดส่วนของ blueprint จริง** ไม่ใช่ข้อสอบจริงที่รั่วไหลมา เนื้อหาบางข้ออาจไม่ตรงกับข้อสอบจริงเป๊ะ ๆ แต่ครอบคลุมหัวข้อย่อยตามที่ blueprint ระบุไว้ทุกหัวข้อ
- หมวด "Cloud Computing" และ "ความสนใจทั่วไปด้านเทคโนโลยี" มีคำถามซ้ำข้ามชุดในสัดส่วนสูงกว่าหมวดอื่น (ดูข้อ 1)
- ข้อ "เลือกหลายคำตอบ" (ms) และ "เขียนตอบ" (written) มีจำนวนคลังจำกัดกว่าข้อปรนัย จึงซ้ำกันได้ระหว่างชุด
- ทุกคำถามผ่านการตรวจสอบอัตโนมัติแล้วว่ามี 5 ตัวเลือกที่ไม่ซ้ำกันในข้อ mc ทุกข้อ (ตรงตามรูปแบบ blueprint)
